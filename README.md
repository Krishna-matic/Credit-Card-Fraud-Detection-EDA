<h2>📌 Project Overview</h2>

<p>
This project performs a comprehensive <b>Exploratory Data Analysis (EDA)</b> on a real-world Credit Card Fraud Detection dataset. The objective is to understand the characteristics of fraudulent transactions, identify the most influential variables related to fraud, and provide business recommendations that support the development of effective fraud detection systems.
</p>

---

<h2>🎯 Objectives</h2>

<ul>
<li>Understand the dataset and its structure.</li>
<li>Perform data cleaning and duplicate analysis.</li>
<li>Analyze transaction amount and transaction time.</li>
<li>Study class imbalance.</li>
<li>Identify important fraud-related features.</li>
<li>Visualize relationships between variables.</li>
<li>Generate business insights and recommendations.</li>
</ul>

<h2>🔍 Key Findings</h2>

<ul>

<li>
The dataset is <b>highly imbalanced</b>, with fraudulent transactions representing only <b>0.17%</b> of all transactions.
</li>

<li>
Most transactions involve relatively low amounts.
</li>

<li>
Fraudulent transactions frequently occur in lower transaction amount ranges.
</li>

<li>
<b>V17, V14, V12, V10, and V11</b> were identified as the most influential features associated with fraud.
</li>

<li>
Combining multiple features provides significantly better fraud separation than relying on a single feature.
</li>

<li>
The dataset exhibits very low multicollinearity, making it suitable for machine learning applications.
</li>

</ul>

<h2>💼 Business Recommendation</h2>

<p>

Based on the analysis, financial institutions should leverage machine learning-based fraud detection systems that utilize multiple transaction features simultaneously. Monitoring should include low-value transactions, as fraudulent activities frequently occur in these ranges. The insights from this project provide a strong foundation for developing accurate fraud detection models.

</p>
